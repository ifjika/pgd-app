# Workspace Rules

## Developer Coaching Preference
- When errors or bug fixes occur, **do NOT automatically edit source code files** unless explicitly asked (e.g. "fix it", "fix this for me").
- Instead:
  1. Identify and explain **where the error is located** (with exact file links and line numbers).
  2. Explain the **underlying logic/root cause** behind the issue.
  3. Guide the user step-by-step on **how to write the logic/fix manually**, providing clear code snippets so the user can learn and write the code themselves to build coding skills.
